How ARP Spoofing Works & How to Fix It

What is ARP Spoofing?
An attacker sends fake ARP (Address Resolution Protocol) replies to associate their MAC address with the IP of another device (usually the gateway). This causes your traffic to route through the attacker's machine instead of directly to the router — enabling eavesdropping, credential theft, and session hijacking.

Signs you're being spoofed:

  • An IP address (especially the gateway) shows multiple MAC addresses
  • Your gateway's MAC address changed from what it was before
  • Internet is slow or connections drop intermittently
  • HTTPS certificate warnings appearing on sites that worked before

Immediate fix (Linux):

# 1. Find your gateway IP and its REAL MAC (check your router's label)
ip route show default
# 2. Flush the poisoned entry
sudo ip neigh flush 192.168.1.1
# 3. Set a static ARP entry (replace with your router's real MAC)
sudo arp -s 192.168.1.1 aa:bb:cc:dd:ee:ff
# 4. Enable kernel-level ARP protection
sudo sysctl -w net.ipv4.conf.all.arp_announce=2
sudo sysctl -w net.ipv4.conf.all.arp_ignore=1
sudo sysctl -w net.ipv4.conf.all.rp_filter=1

Permanent fix:

  • Add the sysctl settings to /etc/sysctl.conf so they persist across reboots
  • Use Dynamic ARP Inspection (DAI) on managed switches
  • Use 802.1X port authentication on your network
  • Use a VPN — encrypted traffic can't be read even if intercepted
  • Install arpwatch: sudo apt install arpwatch — monitors ARP changes 24/7

Find the attacker:

  • Note the attacker's MAC address from the scan results
  • Use the Intruder Trace section below with the attacker's IP
  • Check your router's DHCP lease table to identify the device
  • Run nmap -sn 192.168.1.0/24 to find all devices and their MACs
  • Block the attacker: sudo iptables -A INPUT -m mac --mac-source XX:XX:XX:XX:XX:XX -j DROP
